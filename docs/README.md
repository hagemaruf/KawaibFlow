## Purpose of Each Folder

### handbook/

Contains the project's core engineering standards, architectural
principles, coding conventions, and long-term vision. These documents
are the primary reference for all engineering decisions.

### workflow/

Defines how the team collaborates, including issue lifecycle, branching
strategy, code review process, pull requests, and AI-assisted
development workflow.

### adr/

Architecture Decision Records (ADRs). Each ADR documents a significant
architectural decision, its context, rationale, and consequences.

### architecture/

High-level architecture documentation, diagrams, dependency graphs,
domain models, and technical designs.

### sprint/

Sprint planning, goals, implementation progress, retrospectives, and
sprint deliverables.

### decisions/

Temporary design notes, research, and technical discussions that may
later evolve into ADRs or permanent documentation.

## Documentation Principles

-   The Engineering Handbook is the single source of truth for
    engineering standards.
-   Development Workflow defines the team's operational process.
-   ADRs record significant architectural decisions.
-   Documentation should evolve alongside the codebase.
-   Keep documents concise, accurate, and version controlled.

## Recommended Reading Order

1.  Engineering Handbook
2.  Development Workflow
3.  Architecture Decision Records (ADRs)
4.  Architecture Documents
5.  Sprint Documents

## Contributing

When introducing significant architectural or process changes:

-   Update the relevant documentation.
-   Create or update an ADR if required.
-   Keep documentation synchronized with implementation.
-   Reference GitHub Issues and Pull Requests where appropriate.
